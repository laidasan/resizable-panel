# 繪製 Sequence Diagram 原則

## 流程管理

1. 一次只繪製一個 Sequence Diagram
2. 繪製前先提出疑問和討論，確認範圍與深度
3. 發現高度相似流程時，先討論處理方式再決策

## 共用流程處理

4. 語意上屬於同一分類的共用邏輯，放在該分類目錄下（以 `0.` 開頭命名）
5. 跨分類的共用邏輯，放在 `_shared/` 目錄
6. 判斷是否放 `_shared/` 的依據：該邏輯是否本質上獨立且通用，即使目前只有一個地方使用，未來有合理可能被複用
7. 共用流程以參數化方式撰寫，各 diagram 透過引用 + 標註差異來連結

## 繪製深度

8. 聚焦在「觸發 → 核心邏輯 → 結果分支」，UI 互動的填寫細節不展開，但操作確認後的成功/失敗路徑要呈現
9. 後續流程完全相同時，只畫到呼叫處並標註引用，附上差異對照表
10. 流程很短且結構相似時，各自獨立一份 diagram 比合併用 `alt` 好讀

## 差異處理

11. 當 diagram 引用其他 diagram 時，附上差異對照表（表格形式），說明哪些參數不同、是否影響業務邏輯
