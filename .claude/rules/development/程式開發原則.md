# 程式開發原則

## SOLID 原則

程式開發遵守 SOLID 原則：
- **S**ingle Responsibility — 每個 class / 模組只負責一件事
- **O**pen/Closed — 對擴展開放，對修改封閉
- **L**iskov Substitution — 子類別可替換父類別
- **I**nterface Segregation — 不強迫依賴不需要的介面
- **D**ependency Inversion — 依賴抽象，不依賴具體實作

## Facade 模式

在遵守 SOLID 的前提下，可使用 Facade 模式封裝內部模組，提供使用者好用的 public API。

## OOP

模組以 OOP 概念進行開發（class、封裝、繼承、多型）。

## 代碼檢查

撰寫代碼後，再次檢查代碼有沒有符合 rules 原則。
