# v1 功能規格

> 版本：v1 — 2 Panel + 基礎拖曳
> 建立日期：2026-05-14

---

## 1. 元件結構

v1 提供兩個元件：

- **PanelGroup** — 容器元件，負責管理內部所有 panel 的 layout 與互動
- **Panel** — 內容承載元件，放置在 PanelGroup 內部

v1 限定 **一個 PanelGroup 內恰好 2 個 Panel**，以水平排列方式呈現（左右並排）。

### DOM 結構

Panel 採用雙層 div 結構：
- **外層 div** — 由 PanelGroup 控制，負責 flex 佈局（尺寸分配）。使用者不應直接操作此層的樣式
- **內層 div** — 使用者的內容區域，可自訂樣式與 class。內建 overflow 捲動處理

PanelGroup 的容器 div 強制使用 flex 橫向排列，使用者不可覆寫 `display`、`flex-direction`、`overflow` 等佈局關鍵樣式。

## 2. 可配置項

### PanelGroup 層級

| 配置項 | 說明 | 預設值 |
|--------|------|--------|
| 初始 layout | 指定各 panel 的初始尺寸比例 | 均分 |
| 停用 resize | 停用整組的 resize 互動 | 否 |
| 停用游標管理 | 關閉自動 cursor 樣式 | 否 |
| 命中區域大小 | 拖曳命中區域的像素大小，分粗指標（觸控）和細指標（滑鼠）兩種 | 觸控 20px / 滑鼠 10px |

### Panel 層級

| 配置項 | 說明 | 預設值 |
|--------|------|--------|
| 預設尺寸 | panel 的初始尺寸，支援 % 和 px | 自動均分 |
| 最小尺寸 | panel 可被縮小到的下限 | 0% |
| 最大尺寸 | panel 可被放大到的上限 | 100% |
| 停用 resize | 停用此 panel 的 resize（拖曳到此 panel 邊界無效） | 否 |

## 3. Layout 行為

### 3.1 初始分配

- 有設定預設尺寸的 panel，按預設尺寸分配（受最小/最大約束 clamp）
- 沒有設定預設尺寸的 panel，均分剩餘空間

### 3.2 總和不變式

所有 panel 的百分比加總**永遠等於 100%**。任何操作（拖曳、容器 resize、程式設定）都必須維持此不變式。若計算結果違反，回退到上一個合法 layout。

### 3.3 雙向約束（Clamp）

調整一個 panel 的尺寸時，必須同時檢查：
- 被調整的 panel 是否違反自身的最小/最大約束
- 另一側 panel 是否因連帶變化而違反約束

兩側都不違反時，layout 才生效。

### 3.4 約束衝突

當兩個 panel 的約束組合在數學上無法同時滿足（例如兩個 panel 的最小尺寸加總超過 100%），按 DOM 位置順序處理約束，靠前（左側）的 panel 優先滿足。若怎麼分配都無法滿足 100% 總和不變式，拒絕該操作並回退到上一個合法 layout。

## 4. 單位支援

### 4.1 支援的單位

| 單位 | 格式範例 | 說明 |
|------|---------|------|
| 百分比 | `50`、`"50"`、`"50%"` | 佔容器可用空間的百分比（0-100） |
| 像素 | `"200px"` | 絕對像素值 |

### 4.2 解析規則

- 純數字 → 視為百分比
- 字串無單位或帶 `%` → 百分比
- 字串帶 `px` → 像素

### 4.3 內部模型

所有尺寸在內部統一轉換為**百分比**進行計算。px 值根據容器當前的可用空間轉換為百分比。

### 4.4 混用行為

同一個 PanelGroup 內可以混用 % 和 px。例如左側 panel 設定 `"200px"` 最小尺寸，右側 panel 設定 `"30%"` 最小尺寸。轉換後統一以百分比進行約束計算。

## 5. 拖曳行為

### 5.1 基本流程

拖曳分為三個階段：
1. **按下（pointer down）** — 在命中區域內按下指標，開始拖曳
2. **移動（pointer move）** — 移動指標，即時更新 panel 尺寸
3. **放開（pointer up）** — 釋放指標，結束拖曳

### 5.2 命中區域

v1 不提供可視化的 Separator 元件。拖曳的觸發區域為**兩個 panel 邊界的隱形區域**。

命中區域以 panel 邊界為中心，向兩側延伸：
- **細指標（滑鼠）**：向兩側各延伸一定像素（預設 10px，可配置）
- **粗指標（觸控）**：向兩側各延伸一定像素（預設 20px，可配置）

系統自動偵測指標類型，選擇對應的命中區域大小。

### 5.3 拖曳期間行為

- 拖曳期間**禁止文字選取**，避免拖曳過程中意外選取頁面內容
- 拖曳期間的 pointer 事件綁定在 document 層級，即使指標移出 PanelGroup 範圍也能持續追蹤
- 若拖曳期間偵測到指標已釋放（例如在 iframe 內放開滑鼠），自動結束拖曳

### 5.4 約束即時生效

拖曳過程中，panel 尺寸的每一幀更新都受到最小/最大約束限制。使用者無法將 panel 拖曳超過約束範圍。

## 6. 游標回饋

### 6.1 Hover 狀態

指標進入命中區域（但尚未按下）時，顯示 `col-resize` 游標，提示使用者此處可拖曳。指標離開命中區域後恢復預設游標。

### 6.2 拖曳狀態

拖曳期間，游標設定在 document body 上，確保即使指標移出 PanelGroup 範圍，游標樣式也不會跳回預設。

| 狀態 | 游標 |
|------|------|
| hover（命中區域內） | `col-resize` |
| 拖曳中（無碰到約束） | `col-resize` |
| 拖曳中（碰到一側約束） | 單向箭頭（`e-resize` 或 `w-resize`），指向仍可移動的方向 |
| 停用狀態 | `not-allowed` |

### 6.3 停用游標管理

PanelGroup 可配置關閉游標管理功能，此時系統不會設定任何 cursor 樣式，由使用者自行處理。

## 7. 容器 Resize 行為

### 7.1 監聽容器尺寸變化

PanelGroup 監聽自身容器的尺寸變化（例如瀏覽器視窗縮放、父容器尺寸調整）。

### 7.2 Resize 策略

v1 僅支援 **preserve-relative-size** 策略：
- 容器尺寸變化時，各 panel 的**百分比維持不變**
- 例如：容器從 800px 變為 1000px，panel 佔比 25% 仍為 25%（像素值從 200px 變為 250px）

### 7.3 約束重算與修正

容器尺寸變化後：
1. 以 px 為單位的約束值重新轉換為百分比（因為容器大小變了，相同的 px 值對應的百分比不同）
2. 以新的約束檢查當前 layout 是否仍合法
3. 若不合法，自動 clamp 到合法範圍

### 7.4 隱藏狀態處理

若 PanelGroup 處於隱藏狀態（例如 `display: none`），容器尺寸為 0，此時延遲 layout 計算。等 PanelGroup 變為可見後再執行初始 layout 分配。

---

## 8. 事件通知

### 8.1 Layout 變動中通知

拖曳過程中，**每一幀**尺寸變化時觸發通知。適合用於即時 UI 回饋。

通知內容：當前各 panel 的百分比。

### 8.2 Layout 變動完成通知

拖曳**結束時**觸發一次通知。適合用於持久化儲存或較耗費資源的後續操作。

通知內容：最終各 panel 的百分比。

### 8.3 去重

若 layout 實際上沒有變化（例如拖曳碰到約束停住後持續移動），不重複觸發通知。
